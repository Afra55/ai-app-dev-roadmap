# 第 10 周：简历优化

## 本周目标

将学习成果转化为**可投递的简历内容**，突出 Android 背景 + AI 应用开发的差异化。

---

## 任务清单

- [ ] 填写 [self-assessment.md](self-assessment.md) 诚实对标目标 JD
- [ ] 用 [resume-template-zh.md](resume-template-zh.md) 起草中文简历

- [ ] 按岗位微调技能关键词顺序（AI 应用 / 银行 Android / 国企 Agent）
- [ ] 请他人或 AI 做一次「30 秒项目口述」反馈

---

## 简历结构建议（1 页为主）

1. **基本信息**：姓名 / 联系方式 / GitHub / 城市
2. **求职意向**：AI 应用开发工程师 / Android 开发（AI 方向）等
3. **技能摘要**：Python、RAG、LangGraph、Android、FastAPI、端云协同（按岗位排序）
4. **项目经历**：1 主 + 1 辅（每个 3–4 条 bullet）
5. **工作经历**：Android 开发经验（突出与 AI 项目可迁移能力）
6. **教育背景**

---

## 项目描述公式

```
[动词] + [技术栈] + [解决什么问题] + [可验证结果]
```

**反例**：学习了 LangChain 和 RAG  
**正例**：基于 Chroma + FastAPI 实现笔记 RAG，保存时自动索引，问答返回答案与来源引用

---

## 常见误区

| 误区 | 改进 |
|------|------|
| 堆砌 5 个以上小 Demo | 只保留 1–2 个能演示的项目 |
| 写「精通大模型」 | 写「熟悉 RAG / Agent 应用开发」 |
| 不写 Android 背景 | 强调端侧部署、端云协同优势 |
| 项目无链接 | 附上 GitHub 仓库与 Demo 说明 |
