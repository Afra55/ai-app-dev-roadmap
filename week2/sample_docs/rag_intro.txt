# RAG 简介

RAG（Retrieval-Augmented Generation，检索增强生成）是一种让大语言模型基于外部知识库回答问题的技术。

## 为什么需要 RAG？

1. **减少幻觉**：模型只根据检索到的资料回答，而不是凭空编造。
2. **知识可更新**：更新文档后重新索引即可，无需重新训练模型。
3. **支持私有数据**：企业内部的文档、手册、规范都可以作为知识库。

## RAG 的基本流程

1. 将文档切分成小块（Chunk）
2. 用 Embedding 模型把每个块转成向量
3. 存入向量数据库（如 Chroma）
4. 用户提问时，先把问题转成向量
5. 在向量库中检索最相关的文档块
6. 把检索结果拼进 Prompt，交给 LLM 生成答案

## 适用场景

- 企业内部知识库问答
- 技术文档助手
- 产品说明书查询
- 学习笔记检索
